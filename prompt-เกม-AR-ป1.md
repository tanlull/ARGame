# Prompt: เกม AR จับคู่คำศัพท์ & บวก-ลบเลข สำหรับเด็ก ป.1 (ฉบับเต็ม)

> วาง prompt ทั้งหมดด้านล่างนี้ให้ AI ที่เขียนโค้ดได้ เพื่อสร้างเกมออกมาเป็นไฟล์ `index.html` ไฟล์เดียว
> (นี่คือสเปกของเกมเวอร์ชันสุดท้ายที่สร้างไว้แล้วในไฟล์ `index.html`)

---

ช่วยสร้างเกม AR แบบไฟล์เดียว `index.html` โดยใช้ HTML, Tailwind CSS และ JavaScript ล้วน (ไม่มี build step)

## เป้าหมายและกลุ่มผู้เล่น
เกมการศึกษาสำหรับนักเรียนชั้น **ประถมศึกษาปีที่ 1** (อายุ 6–7 ปี) ฝึกคำศัพท์และการบวก-ลบเลข ผ่านการ "หยิบการ์ดมาจับคู่" ด้วยมือผ่านกล้อง หรือเมาส์/สัมผัส คำสั่งและคำอธิบายทั้งหมดเป็น **ภาษาไทย** ตัวอักษรใหญ่ อ่านง่าย สีสันสดใส

**หัวข้อเกม:** เกมจับคู่มหัศจรรย์ AR — คำศัพท์ & บวก-ลบเลข ป.1

## แนวคิดการเล่น
ผู้เล่น "หยิบ" การ์ดคำตอบที่ลอยอยู่ด้านบน แล้วนำไปวางบน "ช่องโจทย์" ที่ถูกต้องด้านล่าง เช่น หยิบการ์ด `CAT` ไปวางบนช่องรูป 🐱 หรือหยิบการ์ด `7` ไปวางบนช่อง `4 + 3 = ?` เมื่อถูก การ์ดจะล็อกเข้าช่อง ได้คะแนน และมีเสียง/ภาพตอบรับ

## เทคโนโลยี
* **MediaPipe Hands** ตรวจจับมือผ่านเว็บแคม (โหลดจาก CDN)
* **Canvas** ซ้อนทับวิดีโอ วาดการ์ด ช่องวาง เคอร์เซอร์ และโครงมือ ทั้งหมดด้วย `requestAnimationFrame` (draw loop)
* **Pinch gesture** (นิ้วโป้งแตะนิ้วชี้) สำหรับหยิบ/ปล่อยการ์ด
* **Mouse/Touch fallback** — ถ้ากล้องหรือ MediaPipe ใช้ไม่ได้ ต้องเล่นด้วยเมาส์/นิ้วบนจอสัมผัสได้ครบทุกขั้นจนจบเกม
* ภาพจากกล้องแสดงแบบกระจกเงา (mirror) และแปลงพิกัด landmark ให้ตรงกับภาพ mirror

## วิชา (classification) + คลังข้อสอบ
มี **4 วิชา** แต่ละวิชามี **คลังข้อสอบ 50 ข้อ** เก็บในโครงสร้างข้อมูลที่แก้ไข/เพิ่มได้ง่าย:

1. **คำศัพท์อังกฤษ** 🔤 — จับคู่รูป (emoji) กับคำศัพท์อังกฤษ เช่น 🐱 = CAT
2. **คำศัพท์ไทย** 📕 — จับคู่รูป (emoji) กับคำศัพท์ไทย เช่น 🐘 = ช้าง
3. **บวกเลข** ➕ — จับคู่โจทย์บวก กับคำตอบ (ผลบวกไม่เกิน 20)
4. **ลบเลข** ➖ — จับคู่โจทย์ลบ กับคำตอบ (ไม่ติดลบ)

ข้อกำหนดคลังข้อสอบ:
* วิชาคำศัพท์อังกฤษและไทย **ใช้คลังรูปภาพชุดเดียวกัน 50 รายการ** (แต่ละรายการเก็บ `{emoji, คำอังกฤษ, คำไทย}`) ครอบคลุมหมวดสัตว์ ผลไม้/อาหาร ธรรมชาติ และสิ่งของรอบตัว
* วิชาบวกเลขและลบเลข **สุ่มสร้างคลัง 50 ข้ออัตโนมัติ** ด้วยฟังก์ชัน เพื่อให้ได้โจทย์หลากหลาย

## รูปแบบหน้าจอ (flow)
1. **หน้าเริ่มต้น** — ชื่อเกม + ปุ่ม "เข้าสู่เกม"
2. **หน้าวิธีเล่น** — อธิบายการหยิบด้วยมือ/เมาส์ และความหมายสีเคอร์เซอร์ + ปุ่ม "เลือกวิชา →"
3. **หน้าเลือกวิชา** — การ์ดวิชา 4 ใบ (ไอคอน + ชื่อ + คำอธิบาย + "คลัง 50 ข้อ") สร้างจาก array วิชาแบบ dynamic
4. **หน้าเล่นเกม** — มีการ์ดคำตอบลอยด้านบน + ช่องโจทย์ 4 ช่องด้านล่าง + แถบ HUD
5. **หน้าจบเกม** — ข้อความ "สำเร็จ!" + ชื่อวิชา + คะแนนรวม + ปุ่ม "เล่นวิชานี้อีกครั้ง" และ "เลือกวิชาอื่น"

## ระบบรอบและคะแนน
* เลือกวิชา 1 ครั้ง ระบบจะ **สุ่ม 12 ข้อจากคลัง 50 ข้อ** มาเล่น แบ่งเป็น **3 รอบ รอบละ 4 ข้อ** (สุ่มใหม่ทุกครั้งที่เล่น)
* **สำคัญ:** ภายในรอบเดียวกัน คำตอบของการ์ดทั้ง 4 ใบต้อง **ไม่ซ้ำกัน** (โดยเฉพาะวิชาเลขที่โจทย์ต่างกันอาจให้คำตอบเท่ากัน) เพื่อกันการ์ดซ้ำที่ทำให้วางถูกแต่ระบบนับผิด
* จับคู่ถูก 1 คู่ = 25 คะแนน (รอบละ 100, รวม 300)
* จับคู่ผิด **ไม่หักคะแนน** การ์ดลอยกลับที่เดิม ขึ้นข้อความให้กำลังใจ
* เล่นครบ 3 รอบ → "สำเร็จ!" แสดงคะแนนรวม (เต็ม 300) และปุ่มเล่นอีกครั้ง

## การตรวจจับมือให้ "นิ่งและคว้าง่าย" (สำคัญต่อคุณภาพ)
* **กรองตำแหน่งแบบปรับตัว (adaptive smoothing):** ขยับมือนิดเดียวให้กรองหนัก (เคอร์เซอร์นิ่ง ลด jitter) ขยับมาก ๆ ให้ตอบสนองไว (ไม่หน่วง)
* ใช้ **จุดกึ่งกลางระหว่างปลายนิ้วชี้กับนิ้วโป้ง** เป็นตำแหน่งเคอร์เซอร์ (นิ่งกว่าใช้ปลายนิ้วเดียว)
* **Pinch แบบมีช่วงกันสั่น (hysteresis):** เกณฑ์ "เริ่มหนีบ" กับ "ปล่อย" แยกกัน เพื่อไม่ให้สลับจับ/ปล่อยรัว ๆ ตอนมือค้างใกล้เส้นแบ่ง
* **ปรับการหนีบตามขนาดฝ่ามือ** (normalize) เพื่อให้ใช้ได้ทั้งตอนมืออยู่ใกล้/ไกลกล้อง
* **Grace period:** ถ้ากล้องตรวจไม่เจอมือ 1–2 เฟรม การ์ดที่ถืออยู่ต้องไม่หลุดทันที (คงตำแหน่งไว้ชั่วครู่)
* **คว้าแบบแม่นแต่เผื่อ:** คว้าได้เมื่อเคอร์เซอร์อยู่ "บนการ์ด" หรือชิดขอบเล็กน้อย (เผื่อ jitter ~20px) ไม่ใช่คว้าจากที่ไกล ๆ
* ระหว่างหนีบค้างไว้แล้วยังไม่ได้การ์ด ให้ **พยายามคว้าใบที่ใกล้ที่สุดต่อเนื่อง** (กวาดมือเข้าหาการ์ดแล้วคว้าติดเอง)
* การ์ดที่กำลังถูกเล็งให้ **เรืองแสงเหลืองและหยุดลอย** เพื่อเป็นเป้าที่ชัดและอยู่นิ่ง

## UI / การออกแบบ
* สไตล์ **Glassmorphism** (พื้นผิวกระจกฝ้า โปร่งแสง เบลอ)
* การ์ดใหญ่ ตัวอักษรใหญ่ สีสด ตัดกันชัด; ฟอนต์ไทยน่ารักอ่านง่าย (เช่น Google Font "Mali") และต้องย่อขนาดฟอนต์อัตโนมัติให้ข้อความยาว (เช่น ELEPHANT, พระอาทิตย์) หรือสมการ (เช่น `5 + 5 = ?`) พอดีในกรอบไม่ล้นทับช่องอื่น
* รองรับ emoji ในการวาดบน canvas
* มีเสียงตอบรับสั้น ๆ เมื่อจับคู่ถูก (เสียงดี๊ง) และผิด (เสียงตุ๊บเบา ๆ) ด้วย Web Audio
* แสดงสถานะเคอร์เซอร์เป็น 4 สี: 🔵 ฟ้า = ปกติ, 🟡 เหลือง = เล็งโดนการ์ด, 🔴 แดง = วางผิดช่อง, 🟢 เขียว = กำลังถือ/จับคู่สำเร็จ
* แถบ HUD แสดง ชื่อวิชา + รอบที่เล่น (เช่น 1/3), คะแนน, จำนวนคู่ที่จับถูก และปุ่มสลับโหมด "กล้อง/เมาส์"

## ข้อกำหนดทางเทคนิค
* เขียนทุกอย่างในไฟล์ `index.html` ไฟล์เดียว (โหลด Tailwind และ MediaPipe จาก CDN ได้)
* รันผ่าน local server ได้ เช่น VS Code Live Server (กล้องต้องใช้ผ่าน `http://localhost` หรือ `https`)
* ถ้า MediaPipe โหลดไม่สำเร็จ หรือผู้ใช้ไม่อนุญาตกล้อง → สลับไปโหมดเมาส์/สัมผัสอัตโนมัติ และยังเล่นจบเกมได้
* รองรับทั้งจอคอมพิวเตอร์และแท็บเล็ต (responsive) คงความคืบหน้าไว้เมื่อปรับขนาดจอ
* จัดโครงสร้างโค้ดให้ครู **แก้ไขเนื้อหาเองได้ง่าย**: เพิ่ม/แก้คำศัพท์ในคลัง, เพิ่มวิชาใหม่ใน array วิชา, ปรับจำนวนรอบ/ข้อต่อรอบ และปรับค่าความไวการคว้า/หนีบ ได้จากตัวแปรด้านบนของสคริปต์
